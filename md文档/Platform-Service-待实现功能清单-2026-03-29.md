## 1. 测试失败项

### 1.1 非指派人可 report（授权检查 Bug）


| 项目  | 内容                                              |
| --- | ----------------------------------------------- |
| 接口  | `POST /api/mining/v1/refresh-tasks/{id}/report` |
| 预期  | 非被分配的 Miner 调用时返回 403 Forbidden                 |
| 实际  | 返回 200 Success                                  |
| 影响  | 任何 Miner 都可以篡改他人的任务结果                           |


**问题分析**：

协议要求只有被分配的 Miner 才能上报任务结果，服务端应检查调用者身份是否与 `assigned_miner_id` 一致。

## 2. 未实现的协议字段

### 2.1 DataSet 缺失字段


| 字段             | 协议用途                                  | 影响                          |
| -------------- | ------------------------------------- | --------------------------- |
| `dedup_fields` | 指定用于内容去重的字段列表（如 `["post_id"]`）        | 无法按内容去重，只能按 URL 去重          |
| `url_patterns` | 指定合法 URL 模式（如 `["x.com/*/status/*"]`） | 无法校验 URL 合法性，Miner 可提交不相关页面 |


**协议示例**：

```json
{
  "dataset_id": "ds_x_posts",
  "dedup_fields": ["post_id"],
  "url_patterns": ["x.com/*/status/*", "twitter.com/*/status/*"]
}
```

**当前 Swagger 定义**：无这两个字段

---

### 2.2 Submission 缺失字段


| 字段           | 协议用途                    | 影响            |
| ------------ | ----------------------- | ------------- |
| `dedup_hash` | 基于 dedup_fields 计算的内容哈希 | 无法查看/验证内容去重结果 |


**协议要求**：

```
dedup_hash = SHA256(dedup_fields[0] + "|" + dedup_fields[1] + ...)
```

**当前实现**：只有 `normalized_url_hash`（URL 哈希），无内容哈希
