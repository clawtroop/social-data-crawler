#### `social_crawler_run_once`

真实执行过程：

1. 先尝试 `POST /api/mining/v1/repeat-crawl-tasks/claim`
2. 返回 `404 not_found`
3. 插件继续尝试 `POST /api/mining/v1/refresh-tasks/claim`
4. 成功领取到 refresh task：
   - `rfs_2548b107-29be-4261-bf09-cf02fb7ab1a1`
5. crawler 开始抓取任务 URL：
   - `http://example.com/posts/1001`
6. 上游返回 `404`
7. 最终 run 失败，`summary.json` 为：
   - `records_total = 1`
   - `records_succeeded = 0`
   - `records_failed = 1`

本次真实联调产物：

- [summary.json](d:/kaifa/clawtroop/social-data-crawler/output/agent-runs/refresh/rfs_2548b107-29be-4261-bf09-cf02fb7ab1a1/summary.json)
- [task-input.jsonl](d:/kaifa/clawtroop/social-data-crawler/output/agent-runs/refresh/rfs_2548b107-29be-4261-bf09-cf02fb7ab1a1/task-input.jsonl)
- [errors.jsonl](d:/kaifa/clawtroop/social-data-crawler/output/agent-runs/refresh/rfs_2548b107-29be-4261-bf09-cf02fb7ab1a1/errors.jsonl)

结论：

- `repeat-crawl-tasks/claim` 的 `404` 更像是“当前无可领取 repeat task / 当前任务未进入可 claim 状态”
- `refresh-tasks/claim` 已被真实验证为可用
- 服务器当前不是“没有任务”，因为确实成功领取到了 refresh task
- 本次 `run-once` 失败的根因不是插件挂载失败，也不是 claim 接口不可用，而是任务样本中的 URL 不可抓取

补充说明：

- 本次下发的 `http://example.com/posts/1001` 明显属于测试样本 URL
- 因此这次失败更准确地归类为“测试任务样本不可抓取”，不应直接认定为平台任务系统整体故障

## 3. 当前阻塞

当前完整联调仍有以下阻塞项：

- `repeat-crawl-tasks/claim` 仍受服务端任务分配状态影响，当前不能稳定 claim
- refresh task 虽可领取，但下发样本可能是测试死链，导致无法完成 crawler 成功产出与 report
- Epoch 相关链路缺少有效数据，仍无法闭环验证结算

## 4. 差异对照表

| 对比项 | 协议预期 | 当前实际观察 | 当前判断 |
| --- | --- | --- | --- |
| Miner heartbeat | miner 可上线并进入在线集合 | 已真实成功 | 接口可用 |
| Repeat Crawl claim | 被分配 miner 可领取任务 | 当前返回 `404` | 仍依赖服务端分配状态 |
| Refresh claim | 被分配 miner 可领取任务 | 已真实领取到 refresh task | 接口可用 |
| Refresh 执行闭环 | 领取后可抓取并 report | 本轮样本 URL 为测试死链 | 失败点在任务样本质量 |
| crawler 本地执行链路 | 对可抓取 URL 应能产出 records | 可访问 URL 模拟成功 | 本地执行能力可用 |
| Epoch settle | 应可按 epoch 结算 | 仍缺有效 epoch 数据 | 环境未满足 |

## 5. 建议下一步

1. 服务端补一条可真实访问的 refresh task / repeat task 样本 URL
2. 服务端明确 `repeat-crawl-tasks/claim` 在“无可领取任务”场景下的预期状态码
3. 在有可抓取任务样本后，重新跑一次 `social_crawler_run_once`，验证抓取与 report 闭环
4. 继续等待或触发有效 epoch，补测结算链路

## 6. 当前结论

截至 2026-03-29 这轮联调，最准确的结论是：

- 任务系统不是“没任务”
- 任务系统也不是“接口整体坏了”
- 当前问题主要集中在：
  - 部分任务仍依赖服务端当前分配状态
  - 已下发测试样本 URL 不可抓取

换句话说，任务系统目前属于：

- 接口层可用：是
- 任务领取：部分可用
- 任务执行闭环：仍依赖服务端当前任务样本质量
